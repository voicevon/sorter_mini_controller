# 芦笋分拣机控制器架构文档

本文档详细描述了 ESP32 控制器在芦笋分拣机项目中的硬件架构与软件数据流设计。

## 1. 硬件架构设计

系统围绕 ESP32 展开，控制 8 路步进电机及对应的归零（HOME）传感器，具体设计如下：

### 1.1 核心控制器
*   **MCU**: ESP32 (基于 PlatformIO + Arduino Framework)
*   通过 ESP32 强大的运算能力与蓝牙模块，实现对分拣机硬件的高效同步控制以及与上位机/手机端的数据交互。

### 1.2 步进电机同步驱动方案 (8轴)
传统的每个电机独立发脉冲的方式无法实现物理级绝对同步且占用大量引脚。本系统采用了**单脉冲共享与独立屏蔽**的创新设计：
*   **单一共享脉冲 (Shared STEP)**: 所有 8 个 A4988 (或同类驱动器) 的 STEP 输入引脚在硬件上全部并联，由 ESP32 的同一个 GPIO (如 `GPIO 5`) 统一输出脉冲。这意味着只要发脉冲，所有未被屏蔽的电机都具备**完全相同的加减速曲线和速度**，实现绝对同步。
*   **硬件级脉冲屏蔽 (ENABLE)**: 使用 8 个独立的 GPIO 控制 8 路硬件逻辑门。这些引脚用于**放行或屏蔽**到达特定驱动器的 STEP 脉冲。与直接关闭 A4988 的 ENABLE 不同，这种方法让驱动器依然保持通电锁步（维持 Holding Torque），防止机械在重力或惯性下发生自转。

### 1.3 SPI 移位寄存器总线 (DIR 与 HOME 扩展)
由于 8 轴系统需要 8 个 DIR（方向输出）和 8 个 HOME（归零输入），为了节约 ESP32 引脚，系统通过 74HC595（串转并输出）和 74HC165（并转串输入）构建了一条共享时钟和锁存的 SPI 通讯总线。
*   **DIR 方向输出 (74HC595)**: ESP32 通过串行数据引脚下发 8 位方向状态。
*   **HOME 限位输入 (74HC165)**: ESP32 通过串行数据引脚回读 8 位限位开关的触发状态。
*   **通信时序**: 软件实现的全双工模拟 SPI，在向 595 移入数据的同时，同步移出 165 的数据。配合锁存器 (LATCH)，可以在极短的时间内完成全部 8 个方向的更新和 8 个限位的读取。

## 2. 软件与数据流架构

### 2.1 数据来源：蓝牙 (BLE) 通讯
分拣机的“大脑”即决策层，并不在 ESP32 内部，而是依靠外部通讯：
*   **上位机工程 (`phone_sorter`)**: 外部手机系统或上位机（如搭载视觉识别的系统）会根据入口处最新扫描到的芦笋特征（例如长度、粗度），计算出该芦笋应该掉落到几号槽。
*   **蓝牙数据接收**: 手机端通过蓝牙 (BLE) 连接 ESP32，并在每次判定完成后，发送一个 Target ID（目标槽号）。
*   **队列缓冲**: ESP32 蓝牙服务接收到 Target ID 后，会将其推入内部缓冲队列 (`InputQueue`) 等待流水线消费。

### 2.2 8级流水线分拣执行 (Pipeline)
*   **流水线模型**: 系统内部维护一个长度为 8 的数组 `_pipeline[8]`，映射物理分拣机上的 8 级分拣机构。
*   **节拍推演**:
    1.  当新的节拍到来时，内部流水线向前推移一格，从蓝牙队列中取出最新的 Target ID 放入第 1 级（`_pipeline[0]`）。
    2.  控制器遍历流水线的每一级，根据预设的分拣树逻辑（Target ID 到达该级时应该往左转还是往右转，或者不用转），生成对应的 8 位方向数据和 8 位脉冲屏蔽数据。
    3.  通过 SPI 下发方向，通过 GPIO 设定屏蔽状态。
    4.  主发生器（`AccelStepper`）统一执行 90 度的步数。运动完成后，位置清零重置，等待物品自由滑行落位。
    5.  完成一次完美的多电机组合动作。

## 3. 归零机制 (Homing)
在机器启动或出现错位时，系统会执行同步归零：
1.  下发全 0 的方向状态（假设低电平为归零方向）。
2.  主发生器产生恒定负向速度的脉冲，并放开所有电机的脉冲屏蔽。
3.  系统高速循环读取 74HC165 的 HOME 状态。
4.  一旦某级限位开关被触发，立即拉高对应的 ENABLE 引脚屏蔽该电机的脉冲，该电机停转，但其他电机继续寻零。
5.  所有电机均触发限位后，关闭总脉冲，重置原点，进入就绪状态。
